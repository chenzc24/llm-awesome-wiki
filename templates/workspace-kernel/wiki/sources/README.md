# Source Pages

Source pages summarize one source packet, deck, document, or raw source
identity. They are not a replacement for `raw/derived/<source-id>/source.md`.

Each source page should link to the packet or source identity it summarizes.

## Routing

Route content here when the workspace needs a short note about one source,
deck, document, or packet.

Source pages should include:

- source ID or source identity
- packet path or packet reference
- compact summary of what the source contains
- source structure summary
- links to related chapter pages
- important gaps or review notes

Source pages should not:

- repeat all extracted text
- repeat full packet metadata
- list every anchor unless it helps navigation
- become a polished copy of `raw/derived/<source-id>/source.md`

Use `source-page.template.md` after a wiki construction analysis has decided
that a source note should be created or updated.
